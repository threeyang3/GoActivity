# 飞书 Bitable 视图设计清单

> 本文档配合 GoActivity V1.1 字段重塑使用。**GoActivity 不写视图配置**，所有视图由用户在飞书 Bitable 右上 `+ 视图` 手工创建。
> 字段类型迁移完成后，按本文档逐个建视图即可。

## 视图一：活动日历（Calendar）

| 项 | 值 |
|---|---|
| 类型 | 日历视图（Calendar） |
| 名称 | 活动日历 |
| 日期字段 | 开始时间 |
| 标题字段 | 标题 |
| 副标题 | 嘉宾 |
| 颜色规则 | 按"活动类型"上色（见下） |

### 颜色映射

| 活动类型 | 颜色 |
|---|---|
| 讲座 | 蓝色 |
| 演出·放映 | 紫色 |
| 比赛·征稿 | 橙色 |
| 志愿者招募 | 绿色 |
| 普通招募 | 黄色 |
| 普通活动 | 灰色 |
| 非活动 | 浅灰 |

### 过滤

- `状态` 不属于 `{failed_sync, ignored_non_event}`
- `时间状态` ≠ `expired`

### 隐藏字段

摘要、关联原因、置信度、原文链接、更新时间、保留决策、用户置顶

## 视图二：待审核看板（Kanban）

| 项 | 值 |
|---|---|
| 类型 | 看板（Kanban） |
| 名称 | 待审核看板 |
| 分组字段 | 状态 |

### 列顺序（按列拖动排序）

`pending` → `extracted` → `failed_extract` → `failed_sync` → `synced`

### 卡片显示字段

- 主：标题
- 副：活动类型（彩色 chip）、开始时间、地点

### 过滤

- `用户置顶` = `false`（让置顶记录留在"用户置顶"视图里）

### 颜色

- `synced` → 绿
- `pending` → 灰
- `failed_extract` / `failed_sync` → 红

## 操作步骤

1. 打开"活动库" Bitable
2. 右上 `+ 视图` → 选类型 → 配置名称、过滤、排序、分组
3. 按上面 11 个视图逐一创建
4. 把"活动总表"设为默认视图（右键 → 设为默认）
5. 把"活动日历"加到导航栏侧边快捷区
6. 收藏最常用的 3 个：活动日历 / 待审核看板 / 活动总表

## 排障

### 看板视图分组是空
"状态"字段必须是 SingleSelect。检查字段类型。

### 颜色规则不生效
飞书的颜色规则按"字段值"匹配，确认选项名与本文档 §活动类型选项 / §状态选项 / §时间状态选项 一致（含中点 `·`）。

### 日历视图打开是空白
开始时间字段必须是 DateTime 类型且时区为 Asia/Shanghai，参考 `feishu-bitable-setup.md` §5。
